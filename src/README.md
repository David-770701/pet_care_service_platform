# Pet Platform

## Run Locally

Run the following commands from the project root directory.

1. Install dependencies:

```bash
pip install -r src/requirements.txt
```

2. Make sure your local MySQL service is running.

3. Initialize the database schema:

```bash
python src/manage.py migrate
```

4. Start the development server:

```bash
python src/manage.py runserver
```

The project does not require a generated SQLite database file. By default it connects to MySQL database `pet_service_db`.

## Default MySQL Connection

The project uses MySQL by default with these settings:

```text
host     = 127.0.0.1
port     = 3306
database = pet_service_db
user     = root
password = (empty)
```

If your MySQL account is different, set environment variables before running.

PowerShell:

```powershell
$env:MYSQL_HOST="127.0.0.1"
$env:MYSQL_PORT="3306"
$env:MYSQL_DATABASE="pet_service_db"
$env:MYSQL_USER="root"
$env:MYSQL_PASSWORD="your_password"
python src/manage.py migrate
python src/manage.py runserver
```

Command Prompt:

```bash
set MYSQL_HOST=127.0.0.1
set MYSQL_PORT=3306
set MYSQL_DATABASE=pet_service_db
set MYSQL_USER=root
set MYSQL_PASSWORD=your_password
python src/manage.py migrate
python src/manage.py runserver
```

## How To Export SQL

The final submission does not include a separate SQL summary file.
If you want to inspect the complete SQL used by this project, you can generate it directly from the codebase and the local MySQL database.

### 1. Export migration SQL

These commands show the SQL generated by Django migrations, which is useful for seeing how the schema is created and evolved:

```bash
python src/manage.py sqlmigrate pet_core 0001
python src/manage.py sqlmigrate pet_core 0008
python src/manage.py sqlmigrate pet_core 0009
python src/manage.py sqlmigrate pet_core 0013
python src/manage.py sqlmigrate pet_core 0014
python src/manage.py sqlmigrate pet_core 0015
python src/manage.py sqlmigrate pet_core 0016
python src/manage.py sqlmigrate pet_core 0022
python src/manage.py sqlmigrate pet_core 0023
python src/manage.py sqlmigrate pet_core 0024
```

### 2. Export final MySQL table definitions

After the database has been initialized, you can export the final structure of any table with `SHOW CREATE TABLE`.
Example:

```bash
python src/manage.py shell -c "from django.db import connection; c=connection.cursor(); c.execute('SHOW CREATE TABLE pet_core_order'); print(c.fetchone()[1])"
```

Replace `pet_core_order` with other table names such as:

- `pet_core_user`
- `pet_core_merchant`
- `pet_core_service`
- `pet_core_order`
- `pet_core_review`
- `pet_core_vaccine`

### 3. Export the trigger SQL

If you want to inspect the trigger that updates merchant ratings automatically:

```bash
python src/manage.py shell -c "from django.db import connection; c=connection.cursor(); c.execute('SHOW CREATE TRIGGER update_merchant_rating_after_insert'); print(c.fetchone()[2])"
```

Using the commands above, you can reproduce the same categories of SQL that would normally appear in a separate SQL summary document:

- `CREATE TABLE`
- `FOREIGN KEY`
- `UNIQUE`
- `CREATE INDEX`
- `CREATE TRIGGER`
